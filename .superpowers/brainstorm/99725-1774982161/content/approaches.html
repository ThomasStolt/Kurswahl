Architektur-Ansätze

Welcher Ansatz passt am besten für die Kurswahl-App?

A

Monolithisch: Python + HTMX

Ein einziger Python-Dienst (FastAPI) rendert das HTML serverseitig. HTMX übernimmt dynamische Updates ohne separates JS-Framework. Alles in einem Docker-Container.

Vorteile

  • Einfachster Aufbau
  • Ein Container, eine Sprache
  • Python-Optimierungsbibliotheken direkt verfügbar

Nachteile

  • Weniger interaktive Tabellen-Bearbeitung
  • Eingeschränkte UX für CSV-Editor
B ⭐ Empfohlen

FastAPI Backend + React Frontend

Python FastAPI übernimmt CSV-Import, Optimierungsalgorithmus (PuLP/OR-Tools) und Export. React (TypeScript) liefert eine interaktive UI mit editierbarer Tabelle. Docker Compose mit zwei Services.

Vorteile

  • Beste UX für interaktiven CSV-Editor
  • Klare Trennung: Logik vs. UI
  • Optimaler Einsatz von Python für den Algorithmus
  • Gut erweiterbar

Nachteile

  • Zwei Services zu warten
  • Etwas mehr Setup-Aufwand
C

Next.js Full-Stack + Python-Microservice

Next.js (TypeScript) für UI und API-Routen. Separater Python-Dienst nur für den Optimierungsalgorithmus. Drei Container (Next.js, Python, Nginx).

Vorteile

  • Modernes Full-Stack-Framework
  • SSR + interaktive Komponenten

Nachteile

  • Höchste Komplexität
  • Für diesen Use-Case überdimensioniert
  • Drei Container zu koordinieren
